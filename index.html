Pedro's Cheat Sheet

// terminal · git · vs code · docker · npm

Navigation
dir List files in current folder (Windows)
ls List files in current folder (Mac/Linux style)
cd .. Go up one folder level
cd FolderName Go into a folder
cd "Claude Docs\Genesys" Go into a folder with spaces in the name — use quotes
code . Open current folder in VS Code
Git — Basics
git status See what files have changed
git add . Stage all changes ready to commit
git commit -m "your message" Save a snapshot with a description
git push Send commits to GitHub
git pull Get latest changes from GitHub
git log --oneline See recent commits as a clean list
git branch Check which branch you're on — should always be * main
git diff See exactly what lines changed before you commit
git remote prune origin Clean up deleted remote branches from your local list
Git — Cloning & Setup
git clone https://github.com/USER/REPO.git Clone a repo into a new folder
git clone https://github.com/USER/REPO.git . Clone into the current folder (note the dot)
git init Initialise a new Git repo in current folder
git remote add origin URL Connect local folder to a GitHub repo
git fetch Download repo history without changing files
git reset --hard origin/main Force local files to match GitHub — use with care
Git — Branches
git branch See all branches, current one is highlighted
git checkout main Switch to the main branch
git merge master Merge master into current branch
git push origin main Push main branch to GitHub
git branch -d master Delete the master branch locally (after merging)
git push origin --delete master Delete master branch on GitHub too
Docker
docker --version Check Docker is installed and working
docker run hello-world Quick test to verify Docker can run a container
docker run -d -p 8080:80 --name my-nginx nginx Run a test web server in the background on localhost:8080
docker ps See running containers
docker ps -a See all containers, including stopped ones
docker stop my-nginx Stop the nginx test container
docker start my-nginx Start the nginx test container again
docker rm my-nginx Remove the nginx container once you no longer need it
npm / Local Development
npm init -y Create a default package.json in the current folder
npm install live-server --save-dev Install live-server as a dev dependency for local development
npm install Install all packages listed in package.json
npm start Run the local dev server from the start script
Ctrl + C Stop the running dev server in the VS Code terminal
"start": "live-server" Add this to the scripts section in package.json
npm vs Docker
npm = local development Fast, simple, hot reload — best for building and testing quickly in VS Code
Docker = container testing Runs your app in a clean, consistent environment — closer to real deployment
Use npm while building Edit code, refresh browser instantly, iterate fast
Use Docker to verify Check everything works in a fresh environment with no local quirks
npm = speed Best for day-to-day development
Docker = consistency Best for testing how the app will actually run

Tips to remember
★ Quotes matter — any folder with spaces in the name needs quotes: cd "Claude Docs"
★ The dot — a single . means "current folder". So git clone URL . clones into where you already are.
★ Always pull before you push — run git pull first if you've made changes on GitHub directly (like uploading files via browser).
★ VS Code shortcut — once you're in a repo folder in terminal, code . opens it instantly. No clicking around.
★ GitHub Pages URL pattern — your sites live at https://delpedro.github.io/REPONAME/ — needs an index.html to work at root.
★ Always check your branch first — run git branch before starting any session. You should always see * main. If you see * master run git checkout main before touching anything.
★ Docker mental model — an image is a template, a container is a running instance, and Docker is the engine running it all.
★ URLs do not go in terminal — if you run a container on localhost:8080, open that in your browser, not in Command Prompt.
★ Stop local server cleanly — after running npm start, use Ctrl + C in the VS Code terminal to stop it.
★ PWAs need localhost or HTTPS — service workers do not work properly from plain file opening, so always run the app through a local server during development.
